### 项目一：CodeMate——基于RAG的个性化Python编程教学智能体

**【赋能学科】：** Python教学
**【核心概念】：** RAG（检索增强生成）、代码大模型、错误根因分析、学习行为序列建模

- **选题背景与痛点：**
  在Python教学中，学生常遇到编译错误不知所措，或者直接复制AI生成的代码导致“知其然不知其所以然”。传统的通用大模型（如ChatGPT）给出的答案往往过于直接，缺乏引导性。
- **项目思路：**
  1. **构建多源知识库（大数据）：** 收集课程历年的学生代码提交记录、常见错误报错日志、优秀作业代码、以及经典的Python教材（如《Think Python》）。将这些数据进行向量化处理，构建一个垂直领域的教学知识库。
  2. **错误代码智能诊断：** 学生输入报错信息或错误代码片段。系统利用大模型结合知识库，不仅给出修改建议，更要利用**思维链（CoT）** 技术，模拟“苏格拉底式提问”，引导学生自己发现错误。
  3. **个性化出题：** 基于学生的学习进度和薄弱点（数据结构、算法逻辑、语法细节），大模型动态生成针对性的练习题，并利用**代码沙盒**（如Docker或Pyodide）进行自动验证。
  4. **特色功能（进阶）：** 实现 **“代码风格评分”** ，利用大模型模仿PEP 8标准，对代码的可读性、逻辑结构进行评分和重构建议。
- **数据要求：** 至少收集5000+条历史错误日志与对应的修正方案；构建包含教学PPT、书籍、优秀代码片段的非结构化数据集。

### 项目二：AITutor——基于多智能体协作的《人工智能导论》课程助教

**【赋能学科】：** 人工智能导论教学本身
**【核心概念】：** 多智能体系统（Multi-Agent）、课程知识图谱、对抗式验证、概念解构

- **选题背景与痛点：**
  《人工智能导论》涉及数学（线性代数、概率论）、算法（搜索、逻辑回归）和前沿技术（Transformer、Diffusion Model）等多维度内容。学生容易概念混淆，且单一AI很难处理“理论推导”与“代码实现”的混合提问。
- **项目思路：**
  1. **多智能体分工协作：** 构建一个“导师组”：
     - **“数学推导Agent”**：专门负责公式推导、数学原理讲解，利用大模型的代码解释器生成数学公式图表。
     - **“算法实现Agent”**：专注于从零开始手写算法（如K-means, CNN），并解释每一行代码的物理意义。
     - **“概念辨析Agent”**：负责对比易混淆概念（如生成式 vs 判别式；梯度消失 vs 梯度爆炸），利用知识图谱进行可视化展示。
  2. **课程知识图谱构建：** 解析课程大纲和经典教材，利用大模型抽取实体（算法名称、数学家、论文年份、应用场景）及其关系，构建结构化知识库。当学生提问时，系统先定位知识点在图谱中的位置，再调用相应Agent。
  3. **自适应测验生成：** 利用大模型根据布鲁姆认知目标分类（记忆、理解、应用、分析、评价、创造）自动生成不同难度的测试题，并通过学生的回答准确率动态调整后续教学内容。
- **数据要求：** 完整的课程教材PDF、历年期末考试题、教学大纲、经典AI算法伪代码集。

### 项目三：ChipLLM——面向微电子领域的集成电路设计辅助与教学系统

**【赋能学科】：** 微电子/集成电路
**【核心概念】：** 硬件描述语言（HDL）生成、代码-电路转换、时序分析、工艺库数据挖掘

- **选题背景与痛点：**
  微电子专业涉及Verilog/VHDL硬件描述语言、版图设计、时序约束。教学难点在于抽象的逻辑功能如何映射为具体的物理电路。学生写Verilog代码时常犯“综合不可实现”的错误。
- **项目思路：**
  1. **硬件描述语言代码生成与验证：** 利用经过微调的Code LLM（如DeepSeek-Coder或StarCoder），让学生输入自然语言描述（如“设计一个支持异步复位的8位计数器”），系统生成Verilog代码。
  2. **“可综合性”诊断（核心亮点）：** 构建一个包含“可综合代码模式”和“不可综合代码模式”的**大数据集**。大模型不仅要检查语法错误，更要识别出学生代码中哪些部分无法被综合成实际电路（如使用了复杂的延时控制、不恰当的initial语句等），并给出符合工艺库标准的修改建议。
  3. **时序分析与波形图理解：** 上传仿真波形图（VCD文件截图或数据），大模型自动解析波形时序，判断是否存在竞争冒险、建立时间违例等问题，并反向推导出代码中的逻辑缺陷。
- **数据要求：** 收集开源处理器核（如RISC-V）的Verilog代码；收集EDA工具（如Vivado, Quartus）的综合报告与仿真日志，构建“错误-修正”对。

### 项目四：SoftMatterGPT——软物质科学实验方案生成与文献挖掘助手

**【赋能学科】：** 软物质（物理/材料/生物交叉）
**【核心概念】：** 科学文献挖掘、实验方案标准化、分子结构解析、多模态数据处理（文本+图像）

- **选题背景与痛点：**
  软物质（如高分子、凝胶、生物膜）研究依赖大量的实验配方和制备工艺，文献浩如烟海，新手很难快速复现经典实验。且软物质研究常涉及“描述性语言”（如“缓慢滴加”、“剧烈搅拌”），这对AI的理解能力要求极高。
- **项目思路：**
  1. **实验配方知识库构建：** 利用大模型从海量arXiv、Nature Materials等顶刊论文中，自动抽取“实验配方”三元组（材料A、浓度B、工艺C、表征结果D）。构建一个结构化的软物质实验数据库。
  2. **自然语言转实验SOP：** 学生输入研究目标（如“我想合成一种高强度自愈合水凝胶”），大模型检索知识库，生成一份标准操作流程（SOP），包含具体的化学试剂配比、反应温度、时间，并自动生成该实验的安全数据表（SDS）提醒。
  3. **表征结果反推：** 上传显微镜图像（SEM/TEM）或流变学数据图，利用多模态大模型（如GPT-4V或开源的CogVLM）自动识别材料微观结构特征（如“纳米纤维网络”、“多孔结构”），并结合文本知识库，推测当前合成工艺中可能存在的参数偏差。
- **数据要求：** 下载软物质领域近5年的顶刊论文PDF（约500-1000篇），进行OCR与PDF解析，重点提取“实验部分”和“结果讨论”的数据。

### 项目五：LegalMind——基于判例大数据与法律推理的教学辅助系统

**【赋能学科】：** 法学
**【核心概念】：** 法律大模型（Law LLM）、类案检索、判决预测、法律论证逻辑、伦理对齐

- **选题背景与痛点：**
  法学教育强调“案例教学法”和“三段论推理”。学生需要大量阅读判决书来训练法律思维。但通用大模型存在“幻觉”问题，会杜撰虚假的法条或判例，且难以处理复杂的事实与规范之间的涵摄过程。
- **项目思路：**
  1. **高质量法律数据集构建：** 收集公开的裁判文书网数据、法律法规库、法学核心期刊论文。重点进行**“案情事实”与“判决结果”的对齐标注**。
  2. **法律论证生成与纠偏：** 输入一个虚拟或真实的案情描述，系统利用大模型模拟“法官”或“双方律师”的角色，生成严格遵循“大前提（法条）-小前提（事实）-结论”结构的法律意见书。
  3. **类案检索与比对：** 利用向量数据库技术，输入当前案情，系统自动检索历史上最相似的10个判例，并利用大模型生成一份“类案比对报告”，分析当前案情与过往判例在关键事实要素上的异同，辅助学生理解“同案同判”的原则。
  4. **伦理与对齐（安全机制）：** 针对法学教育的特点，该项目必须包含**对抗性测试模块**。即构建“恶意提问”数据集（如“如何规避法律责任”），训练模型识别并拒绝回答不符合法律职业道德的问题，引导学生在合规范围内使用AI。
- **数据要求：** 需使用清洗后的结构化裁判文书数据（建议使用公开数据集，如“中国法律人工智能数据集”），注意数据隐私和合规性。
