## 3. 数据持久化加固

### 3.1 根因分析

`frontend/user_data.py` 的 `加载用户数据()` 函数中：

```python
def _读文件(文件名):
    fp = d / 文件名
    if fp.exists():
        try:
            return json.loads(fp.read_text())
        except Exception:
            return []  # ← 静默吞异常，返回空列表
    return []
```

当 JSON 文件因以下原因读取失败时，返回空数组：
- 文件过大导致内存不足
- 磁盘 I/O 错误
- JSON 格式损坏（写入中途崩溃）
- 进程被 kill 导致写入不完整

后续 `保存用户数据()` 将空数组覆盖写回磁盘 → **数据永久丢失**。

### 3.2 修复方案

#### 原子写入（Atomic Write）

```python
def _原子写入(文件路径: Path, 数据) -> None:
    tmp = 文件路径.with_suffix(文件路径.suffix + ".tmp")
    tmp.write_text(json.dumps(数据, ensure_ascii=False, indent=2))
    tmp.replace(文件路径)  # os.rename 是原子操作
```

#### 自动备份

```python
def _带备份写入(文件路径: Path, 数据) -> None:
    if 文件路径.exists():
        bak = 文件路径.with_suffix(文件路径.suffix + ".bak")
        文件路径.replace(bak)  # 旧文件 → .bak
    _原子写入(文件路径, 数据)
```

#### 读取容错

```python
def _安全读文件(文件路径: Path):
    if not 文件路径.exists():
        return []
    try:
        data = json.loads(文件路径.read_text())
        if not isinstance(data, list):
            return []  # 格式不对但保留现场
        return data
    except Exception as e:
        # 写入错误日志
        _写错误日志(f"读取 {文件路径} 失败: {e}")
        # 尝试从 .bak 恢复
        bak = 文件路径.with_suffix(文件路径.suffix + ".bak")
        if bak.exists():
            try:
                return json.loads(bak.read_text())
            except Exception:
                pass
        # 保留现场，不覆盖
        raise  # 向上抛出让调用方处理
```

#### 文件大小检查

- 保存前检查：单个 JSON > 10MB 时在前端给用户警告（但正常保存）
- `加载用户数据()` 返回时附带 `_warnings` 字段，提示用户错题数过多

### 3.3 影响范围

- `frontend/user_data.py`：`保存用户数据()` 改为原子写入 + 自动备份
- `frontend/pages/quiz.py`：调用方无需改动（接口兼容）
- `frontend/app.py`：调用方无需改动

## 4. 一键安装脚本

### 4.1 功能清单

| 步骤 | macOS/Linux (`install.sh`) | Windows (`install.bat`) |
|------|--------------------------|------------------------|
| 1 | 检测 Python 3.11+ | 检测 Python 3.11+ |
| 2 | 创建虚拟环境 `.venv` | 创建虚拟环境 `.venv` |
| 3 | 激活 + pip install | 激活 + pip install |
| 4 | 交互输入 API Key | 交互输入 API Key |
| 5 | 写入 `.env` | 写入 `.env` |
| 6 | 生成桌面快捷方式 | 生成桌面快捷方式 |
| 7 | 打印完成信息 | 打印完成信息 |

### 4.2 文件位置

- `aitutor/install.sh` — macOS/Linux 安装脚本
- `aitutor/install.bat` — Windows 安装脚本

### 4.3 快捷方式

**macOS**: 生成 `~/Desktop/AITutor.command`，内容为：
```bash
#!/bin/bash
cd "$(dirname "$0")/../aitutor" && source .venv/bin/activate && bash run.sh
```

**Windows**: 生成 `%USERPROFILE%\Desktop\AITutor.bat`，内容为：
```bat
@echo off
cd /d %~dp0..\aitutor && call .venv\Scripts\activate.bat && bash run.sh
```

### 4.4 错误处理

- Python 版本不够 → 提示升级并给出下载链接
- pip install 失败 → 提示检查网络并重试
- API Key 跳过输入 → 生成 `.env` 模板，用户可稍后手动填写
